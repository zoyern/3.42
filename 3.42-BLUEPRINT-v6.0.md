### Types comme contraintes, non représentations

**✅ Paradigme innovant**

Un type **n'est pas une étiquette** — c'est une **contrainte explicite sur les valeurs** :

```3.42
; Pas : "x est de type entier"

name : ".*"           ; Type = pattern regex

```

**Avantage** : impossible d'avoir une température négative, une probabilité > 1, ou un âge > 150. Le type empêche les erreurs à la compile, pas à l'exécution.

### Sécurité mémoire sans complexité

**⚠️ Prototypage avancé**

La plupart des langages modernes offrent :

**3.42 utilise les arènes** (arenas) :


### Preuve de Personnalité (Proof of Personhood)

**💡 Recherche en cours**

Comment prouver que vous êtes humain sans révéler qui vous êtes ?

Approche :

1. **Biométrie zéro-connaissance (Zero-knowledge proof)** : scanner iris → hash → preuve que hash=hash, sans révéler iris
2. **Preuve de vie** : défi aléatoire qu'une IA ne peut pas passer (par ex., reconnaître un visage de côté)
3. **Graphe social (Social graph)** : attesté par amis/collègues (Web of Trust)

**Résultat** : "Je prouve que je suis humain" sans révéler d'identité personnelle.

### Système de clés cryptographiques

**✅ Cryptographie établie**

Chaque DID a 2 clés :

```

  ↓
  [Algorithme: EdDSA ou ECDSA]
  ↓
Clé publique (Public key) : partagée librement
  ↓
  [Utilisée pour vérifier signatures]
```

Exemple de signature :

```3.42

message = "J'ai créé ce code le 2026-03-02"

; Bob vérifie la signature (sans connaître la clé privée)
verify(message, signature, alice_public_key) → ✅ Valide
```
