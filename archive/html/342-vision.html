Type System — Constraint as First-Class

TYPE = CONTRAINTE, pas représentation

Types are constraints discovered by range analysis, not labels. The IDE tells you which type fits.

← number
├─ integer
│ ├─ i8, i16, i32, i64, i128
│ ├─ u8, u16, u32, u64, u128
│ └─ bigint (unbounded)
├─ rational
│ └─ p/q (exact fractions)
├─ decimal
│ └─ fixed-point arithmetic
├─ float
│ ├─ f16, f32, f64, f128
│ └─ bfloat16 (ML friendly)
└─ complex
└─ (real, imag) pair
x : 0..100
→ IDE: u8 (1 byte, range check at compile time)

y : -1000..1000
→ IDE: i16 (2 bytes, signed)

z : 0..(10^18)
→ IDE: u64 (8 bytes, unsigned)

Key Innovation: Automatic Type Discovery

  • ✓ No explicit type annotations needed for numeric ranges
  • ✓ Compiler infers smallest safe type for each variable
  • ✓ Memory-optimal: x : 0..255 automatically becomes u8
  • ✓ Zero runtime overhead: all decisions made at compile time
  • ✓ IDE shows you the type in real-time as you type constraints

Memory Model — Pauli Exclusion + FIFO Channels

342 Memory
MOVE by default — Pauli exclusion: one owner at a time
@ read — Borrow for reading (shared, immutable)
@@ write — Exclusive borrow (mutable access)
Simple semantics — "It's yours until you give it away"
No complex borrow checker needed
Arena Scoping
{} = arena scope — Creates a bounded memory region
FIFO channels — Values flow through pipes in order
Exit = FREE — Automatic deallocation at scope end
Deterministic — Know exactly when memory is freed
Like stack allocation, but composable
vs. Alternatives
Rust: Complex borrow rules, learning curve steep
Go: Garbage collection, unpredictable pauses
342: Simple + Safe by design, Pauli physics-inspired
Physics first, then safety
Memory Safety Without Sacrifice
No null pointers. No buffer overflow. No use-after-free.
No double-free. No data races. All guaranteed at compile time.
And the syntax is simpler than Rust.

Development Roadmap

Phase 1
PEG Grammar + Parser
Formalize the complete grammar using Parsing Expression Grammar (PEG). Build a robust recursive descent parser that handles all 18 bosons, 5 gluons, and separator rules. Comprehensive error reporting and recovery.
Phase 2
AST → ParticleIR
Convert Abstract Syntax Trees into ParticleIR (Intermediate Representation). Implement type inference, constraint solving, and the range analysis system. Validate memory safety properties.
Phase 3
Cranelift POC
Proof of concept: compile to native code via Cranelift. Target x86-64 and ARM64. Get "hello world" running. Benchmark against C/Rust baseline. Verify memory safety properties at runtime.
Phase 4
10 Example Programs
Implement 10 diverse programs: sorting, file I/O, networking, numeric computation, tree traversal, etc. Stress-test the type system, memory model, and core features. Gather real-world feedback.
Phase 5
Sugar System
Implement syntactic sugar: loops, pattern matching, list comprehensions, etc. All sugar expands to core composition rule. Keep the language orthogonal and learnable without hidden complexity.
Phase 6
WASM + Web Deployment
Compile ParticleIR to WebAssembly. Deploy browser-based IDE and online playground. Enable collaborative learning and rapid prototyping. Make 3.42 accessible to anyone with a web browser.
